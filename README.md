# Price Monitor

Мониторинг цен маркетплейсов (Ozon, в перспективе Wildberries): проверка цен, алерты в Telegram и e-mail, хранение истории, контроль отклонений.

Проект в разработке.

---

## Запуск вручную

Из корня проекта (каталог, где лежат `main.py`, `config/`, `docs/`):

```bash
python main.py
```

Перед запуском: создать `config/accounts.json`, настроить `.env` (см. ниже). Файл `config/accounts.json` должен существовать до первого запуска; при его отсутствии приложение завершится с ошибкой.

**Отчёт Price Intelligence (последние 24 ч):** из корня проекта запустить `python report_price_intelligence.py` — выводится текстовая сводка: топ изменений цен, самые активные SKU, возможные аномалии. Используются только данные из `price_history`; мониторинг (main.py) не задействован.

**Как создать .env:** скопировать `.env.example` в `.env` и подставить свои значения. Файл `.env` не коммитить.

## Конфигурация аккаунтов (config/accounts.json)

Поддерживается **несколько аккаунтов** в одном запуске. Файл — JSON с массивом `accounts`.

**Формат элемента:**
- `marketplace` — `"ozon"` или `"wildberries"`.
- `name` — **уникальное** имя аккаунта (используется как идентификатор в кэше и БД; не должно повторяться).
- Для **Ozon**: `client_id_env`, `api_key_env` — имена переменных в `.env`, из которых подставляются ключи API.
- Для **Wildberries**: `api_key_env` — имя переменной в `.env` с API-ключом.

**Пример (несколько аккаунтов Ozon):**

```json
{
  "accounts": [
    {
      "marketplace": "ozon",
      "name": "Ozon #1",
      "client_id_env": "OZON_CLIENT_ID_1",
      "api_key_env": "OZON_API_KEY_1"
    },
    {
      "marketplace": "ozon",
      "name": "Ozon #2",
      "client_id_env": "OZON_CLIENT_ID_2",
      "api_key_env": "OZON_API_KEY_2"
    }
  ]
}
```

В `.env` задать соответствующие переменные. Для **Wildberries** используется один API-токен с правами **Promotion** (карточки товаров, cards/list) и **Цены и скидки** (list/goods/filter). Реализована пагинация cards/list (каталоги &gt;100 карточек) и разбиение запроса цен на чанки при nmList &gt; 1000.

### Переменные окружения (.env)

**Обязательные (для работы мониторинга и алертов):**
- `TELEGRAM_TOKEN` — токен бота Telegram
- `TELEGRAM_CHAT_IDS` — идентификаторы чатов (через запятую)
- `OZON_CLIENT_ID_1`, `OZON_API_KEY_1` — ключи Ozon API для аккаунта (имена в config/accounts.json могут отличаться; поддерживается старый формат `OZON1_CLIENT_ID`, `OZON1_API_KEY`)

**Опциональные:**
- `SEND_TELEGRAM_ALERTS` — включить/выключить отправку алертов в Telegram (`true`/`false`)
- `SEND_EMAIL_ALERTS` — включить/выключить отправку алертов по e-mail (`true`/`false`)
- `SEND_STARTUP_MESSAGE` — отправлять стартовое сообщение в Telegram (`true`/`false`)
- `SEND_STARTUP_EMAIL` — отправлять стартовое письмо на e-mail (`true`/`false`)
- `SEND_RUN_SUMMARY` — отправлять ли итог запуска в Telegram (`true`/`false`, по умолчанию не отправлять)
- `DRY_RUN` — режим без отправки в Telegram и без записи в БД (`true`/`false`)
- `EMAIL_ENABLED` — включить e-mail канал (`true`/`false`)
- `EMAIL_SMTP_HOST`, `EMAIL_SMTP_PORT`, `EMAIL_SMTP_USER`, `EMAIL_SMTP_PASSWORD` — SMTP-подключение
- `EMAIL_USE_TLS` — использовать STARTTLS (`true`/`false`)
- `EMAIL_FROM` — адрес отправителя
- `EMAIL_TO` — список получателей через запятую
- `EMAIL_SUBJECT_PREFIX` — префикс темы (например, `[Price Monitor]`)

Итог запуска в Telegram отправляется **только** при `SEND_RUN_SUMMARY=true` **и** `DRY_RUN=false`. Лог Run summary в файл пишется всегда.

- **Алерты (опционально)**:
  - LAST_PRICE_ALERT_THRESHOLD_PERCENT — порог изменения относительно последней цены (по умолчанию 10%)
  - DAY_START_ALERT_THRESHOLD_PERCENT — порог изменения относительно цены на начало дня (по умолчанию 20%)
  Алерт срабатывает, если абсолютное изменение превышает указанный порог.
  - `ALERT_COOLDOWN_MINUTES` — пауза в минутах перед повторным алертом по тому же SKU/цене (по умолчанию: 60).
  - `MAX_ALERT_CHANGE_PERCENT` — не слать алерт при скачке выше этого % (по умолчанию: 100).
- **Уведомления**: стартовое сообщение в Telegram управляется `SEND_STARTUP_MESSAGE`, стартовое письмо — `SEND_STARTUP_EMAIL`.

### Dry Run mode

Режим «сухого прогона»: приложение выполняет все шаги (загрузка продуктов, цен, анализ), но **не отправляет** уведомления в Telegram, **не сохраняет** цены в БД и **не обновляет** состояние cooldown алертов. В лог пишутся сообщения вида `DRY RUN: alert detected but not sent` и `DRY RUN: prices not saved to database`.

Включение: в `.env` задать

```
DRY_RUN=true
```

По умолчанию режим выключен (`false`).

## Регулярный запуск (планировщик)

Рекомендуемый способ — **внешний планировщик**: приложение не содержит встроенного планировщика, каждый запуск выполняет один проход и завершается. Так проще управлять расписанием и логами.

- **Рабочий каталог** при запуске должен быть **корень проекта** (иначе не найдутся `config/accounts.json`, `prices.db` и т.д.).
- Интервал между запусками лучше делать больше типичного времени выполнения (например, не чаще чем раз в 5–15 минут).
- Защита от двойного запуска: при старте создаётся файл `price_monitor.lock`; если он уже есть, новый процесс пишет в лог предупреждение и завершается. После окончания работы (в т.ч. при ошибке) блокировка снимается. Если процесс был принудительно завершен, файл может остаться — его можно удалить вручную.

### Windows (Task Scheduler)

1. Открыть «Планировщик заданий» (taskschd.msc).
2. Создать задачу: «Создать задачу» (не «Простая задача»).
3. Вкладка **Общие**: имя, например «PriceMonitor»; выполнять для текущего пользователя.
4. Вкладка **Триггеры**: «Создать» → расписание (например, каждые 15 минут или по расписанию).
5. Вкладка **Действия**: «Создать» → действие «Запуск программы»:
   - Программа: `python` (или полный путь к `python.exe`);
   - Аргументы: `main.py`;
   - Рабочая папка: каталог проекта (где лежит `main.py`).
6. Сохранить. При необходимости задать переменные окружения в задаче или запускать через `.env` в каталоге проекта.

### Linux / macOS (cron)

Ввести в crontab (`crontab -e`), подставив свой путь к проекту:

```cron
# каждые 15 минут
*/15 * * * * cd /path/to/Pricemonitor && python main.py >> logs/cron.log 2>&1
```

Либо без перенаправления (вывод в почту или журнал cron):

```cron
*/15 * * * * cd /path/to/Pricemonitor && python main.py
```

Каталог `logs/` при необходимости создать заранее.
